<Info>If SelfGuard was instantiated with a key pair, it will automatically encrypt data such that only the owner of the key pair will be able to decrypt the data.</Info>

## AES-256 Encryption
This allows you to encrypt/decrypt any piece of data. Encryption keys are stored with SelfGuard.

### Encrypt (AES-256)

<ParamField path="value" required={true} type="object | string">
 A string or object that you want encrypted.
</ParamField>

```js
await sg.encrypt(value);
```

### Decrypt (AES-256)

<ParamField path="value" required={true} type="object | string">
 A string or object that you want decrypted.
</ParamField>

<ParamField path="encryption_key_id" required={true} type="string">
 The id of the encryption key returned from `sg.encrypt`.
</ParamField>
```js
await sg.decrypt(value, encryption_key_id);
```

## Password Encryption
This allows you to encrypt/decrypt any piece of data with a password. Nothing is stored with SelfGuard here.

### Encrypt With Password
<ParamField path="value" required={true} type="object | string">
 A string or object that you want encrypted.
</ParamField>
<ParamField path="password" required={true} type="password">
 The password you want to use to encrypt the value.
</ParamField>
```js
sg.encryptWithPassword(value,password);
```

### Decrypt With Password

<ParamField path="value" required={true} type="object | string">
 A string or object that you want decrypted.
</ParamField>
<ParamField path="password" required={true} type="string">
 The password you want to use to decrypt the value.
</ParamField>
```js
sg.decryptWithPassword(value, password);
```

## Data Tokenization
This allows you to encrypt data and store the encrypted data with SelfGuard itself. Each piece of data encrypted will have a respective token id.

### Tokenize

<ParamField path="value" required={true} type="object | string">
 A string or object that you want tokenized.
</ParamField>
```js
await sg.tokenize(value);
```

### Detokenize

<ParamField path="token_id" required={true} type="string">
 The token id of the data previously tokenized using `sg.tokenize`
</ParamField>
```js
await sg.detokenize(token_id);
```


## Encrypted Key/Value Storage
This allows you to store any key value data where the value is encrypted.

### Put

<ParamField path="key" required={true} type="string">
 The key with which you want to associate the value.
</ParamField>
<ParamField path="value" required={true} type="object | string">
 A string or object that you want encrypted.
</ParamField>
```js
await sg.put(key, value);
```

### Get

<ParamField path="key" required={true} type="string">
 The key with which you want to retrieve the value.
</ParamField>
```js
await sg.get(key);
```

### Get All Keys

```js
await sg.getKeys();
```

## Encrypted Array Storage

Used as an encrypted database to store key -> multiple values. Value data is fully encrypted by an encryption key set up at the initiation of the array. Value data can only be decrypted by users who have been assigned access to the encryption key via asymmetric encryption.

### Create Array

<ParamField path="name" required={true} type="string">
 The name of the array you want to create.
</ParamField>
```js
sg.createArray(name);
```

### Add To Array

<ParamField path="name" required={true} type="string">
 The name of the array you created.
</ParamField>
<ParamField path="value" required={true} type="string">
 The value that you want to encrypt and add to the array.
</ParamField>

```js
await sg.addToArray(name, value);
```

### Get Array
<ParamField path="name" required={true} type="string">
 The name of the array you created.
</ParamField>
```js
await sg.getArray(name);
```

### Get Array Names

```js
await sg.getArrayNames();
```
